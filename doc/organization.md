# Team Organization and Roles

This document describes the responsibilities and contributions of each team member for the Foodie Core project. Each member is responsible for implementing a core feature of the application, while also collaborating on integration, testing, and final polish.

---

## Seth Payne — Favorites System

Seth Payne is responsible for designing and implementing the Favorites functionality. This includes adding a save/favorite button to recipes, creating a Favorites page that lists saved recipes, and allowing users to remove favorites. The Favorites system uses browser-based storage (localStorage) to ensure that saved recipes persist across page refreshes and browser restarts. Seth also assists with navigation integration and overall application testing.

---

## Heidi Ganim — Recipe Search

Heidi Ganim is responsible for implementing the recipe search functionality. This includes building the search interface, handling user input, displaying a list or grid of recipe results, and ensuring each result displays the recipe name, image, and a way to navigate to the detailed recipe page. Heidi ensures that searching common terms returns multiple relevant results.

---

## Dominiq Marica — Recipe Details

Dominiq Marica is responsible for the recipe details page. This includes displaying detailed information for a single recipe such as the title, image, ingredients list, and step-by-step instructions. Dominiq ensures that clicking a recipe from the search results opens the correct detailed view.

---

## Brayton Rumple — Reviews System

Brayton Rumple is responsible for implementing the reviews feature. This includes displaying existing reviews on the recipe details page and providing a form for users to submit new reviews. Reviews include a name, rating (1–5), and written comment. Brayton ensures reviews are correctly associated with each recipe and displayed after submission.

---

## Shared Responsibilities

All team members collaborate on:
- Navigation between pages
- Ensuring features integrate correctly
- Maintaining consistent styling
- Testing and debugging
- Final review and submission
